## Critical Lessons Learned This Session

### The Orchestrator Does Not Do The Work
The orchestrator reads state, decides what needs doing, decides which agent does it, and manages the handoff. It does NOT pick up the wrench. The orchestrator.md role file currently says "fix blockers directly" — this is WRONG and needs to be updated to say "dispatch the appropriate agent."

### Ray Does Not Use The Terminal
Ray opens agent chat sessions and talks. The orchestrator's job is to produce prompts that Ray can hand to other agents — or better, to invoke them as subagents. Never give Ray a shell command.

### The Hook Exists and Works
`.githooks/pre-commit` syncs: architecture/ → docs/architecture/, .claude/agents/ → docs/agents/, Templates/ → docs/sawmill-templates/, CLAUDE.md → docs/institutional-context.md, sawmill/COLD_START.md → docs/sawmill/COLD_START.md. Most audit "stale" findings are just "you didn't commit."

### The Mirror Rule
Source of truth is ALWAYS the source location. Never the docs/ copy. The hook enforces this at commit time.
- `.claude/agents/*.md` → source of truth for roles
- `architecture/*.md` → source of truth for authority docs
- `Templates/*.md` and `Templates/*.yaml` → source of truth for templates
- `CLAUDE.md` → source of truth for institutional context
- `docs/` is ALWAYS a derived mirror, never edited directly

### Don't Over-Engineer Simple Things
A role file IS the prompt. "You are the auditor" is the entire invocation. Stop wrapping things in run.sh flags, scripts, and infrastructure when a sentence works.
