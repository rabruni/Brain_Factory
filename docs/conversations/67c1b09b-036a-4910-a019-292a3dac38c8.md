# Auto-Sync: Brain Factory Repo → Backstage TechDocs

## Context

The Backstage Coverage Audit found ~68 repo files invisible in Backstage. The root causes:
1. **Manual nav** — mkdocs.yml has a 90-line hand-maintained `nav:` block. Adding a file requires editing it.
2. **Manual publish** — `mkdocs build` + `cp -r` to Backstage static dir is done by hand.
3. **Narrow sync** — the existing `hooks/sync_docs.py` only covers `architecture/` and `Templates/`. Everything else (YAML registries, Python scripts, spec packs, prompts, staging code) is invisible.
4. **Non-Markdown files invisible** — `.py`, `.yaml`, `.sh`, `.txt` files can't render as TechDocs pages without a Markdown wrapper.

Ray wants: everything in the repo visible in Backstage, automatic change detection, and the ability to see code (Python, YAML) directly.

## Current Mechanisms (keep working, don't break)

| Mechanism | File | What it does |
|-----------|------|-------------|
| Pre-commit mirror sync | `.githooks/pre-commit` → `sawmill/sync_portal_mirrors.py` | Copies PORTAL_MAP `sync: mirror` sources → docs/ on commit |
| MkDocs build hook | `hooks/sync_docs.py` | Hardlinks `architecture/` and `Templates/` into `docs/`, auto-generates nav — **but NOT wired up** (no `hooks:` entry in mkdocs.yml) |
| PORTAL_MAP | `docs/PORTAL_MAP.yaml` | 44 entries declaring which files mirror where |
| Backstage config | `dopejar/app-config.yaml` | `builder: 'local'`, `publisher: type: 'local'`, `generator.runIn: 'local'` |
