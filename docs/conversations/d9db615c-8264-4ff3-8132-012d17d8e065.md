Here's the full picture:

## Is Docker running?

**No.** The Docker daemon is not running. The socket at `~/.docker/run/docker.sock` doesn't exist. Docker Desktop (the macOS app) is installed but not open.

## What does it provide?

Four services for the DoPeJarMo runtime (`dopejar/docker-compose.yml`):

| Service | Image | What it does |
|---------|-------|-------------|
| **kernel** | Custom build (`./kernel`) | The DoPeJarMo core — HO1, HO2, HO3 (Graph), Write Path, WebSocket on :8080 |
| **ledger** | `codenotary/immudb` | Append-only hash-chained truth store (the Ledger primitive), gRPC on :3322 |
| **zitadel** | `ghcr.io/zitadel/zitadel` | Identity + authorization (OIDC, JWT, agent permissions), API on :8085 |
| **zitadel-db** | `postgres:17-alpine` | PostgreSQL backing store for Zitadel |

Plus **Ollama** runs host-native (not in Docker) for Metal GPU acceleration — kernel reaches it via `host.docker.internal:11434`.

## What's required to start it?

Open **Docker Desktop** (the app in `/Applications/Docker.app`). Once the daemon is running, then `docker compose up` from `/Users/raymondbruni/dopejar/` brings up the stack.

## Why doesn't it survive a reboot?

Docker Desktop for Mac is a **GUI application**, not a system daemon. The two LaunchDaemons that do exist (`com.docker.socket` and `com.docker.vmnetd`) are just privileged helpers for socket forwarding and networking — they don't start the Docker engine itself.

The Docker engine runs inside Docker Desktop's Linux VM (managed by the app). That VM only starts when you open Docker Desktop. Apple doesn't auto-launch GUI apps on boot by default — you'd need to add Docker Desktop to **System Settings > General > Login Items** to make it start automatically on login.

**To make it survive reboots:** System Settings > General > Login Items > add Docker. Or in Docker Desktop: Settings > General > "Start Docker Desktop when you sign in."

Open **Docker Desktop**:

1. Press **Cmd + Space**, type `Docker`, press Enter.

That starts the daemon. Wait for the whale icon in your menu bar to stop animating (about 30 seconds).

**To make it survive reboots:**

Once Docker Desktop is open:

1. Click the whale icon in the menu bar
2. Click **Settings** (gear icon)
3. Under **General**, check **"Start Docker Desktop when you sign in to your computer"**
4. Click **Apply & Restart**

That's it. Docker will auto-start on every login from now on.
