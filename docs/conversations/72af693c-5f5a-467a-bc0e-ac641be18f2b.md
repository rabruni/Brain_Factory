# Plan: Runtime Role Enforcement Hooks for Sawmill (Hybrid Env + Sentinel)

## Context

Sawmill agents can drift outside their file-ownership lanes. The orchestrator (HO2) is dispatch-only by instruction, but nothing enforces it at runtime. Claude Code has built-in hooks that fire on every tool call — we use them to hard-deny Write/Edit outside each role's lane.

Two runtime paths need enforcement:
- **Headless** (`run.sh`): env vars set per-process, no race on parallel Turn B/C
- **Conversational** (Ray talks to orchestrator, orchestrator dispatches via Agent tool): sentinel file written by orchestrator before each dispatch

## Files

| File | Action |
|------|--------|
| `.claude/hooks/sawmill-guard.sh` | Create — PreToolUse, denies Write/Edit outside role's lane |
| `.claude/hooks/sawmill-state.sh` | Create — PreToolUse, injects pipeline state for orchestrator |
| `.claude/hooks/sawmill-stop.sh` | Create — Stop, blocks orchestrator quit when dispatch has no result |
| `.claude/settings.json` | Create — project-level hook wiring |
| `sawmill/boot-orchestrator.sh` | Create — optional launcher, sets env var + sentinel |
| `sawmill/run.sh` | Edit — export `SAWMILL_ACTIVE_ROLE` + `SAWMILL_ACTIVE_FMWK` for Claude invocations |
| `.claude/agents/orchestrator.md` | Edit — add Dispatch Protocol section (sentinel write before/after Agent tool) |
| `.gitignore` | Edit — add `sawmill/.active-role` |

## Role Resolution (all hooks)

```bash
ROLE="${SAWMILL_ACTIVE_ROLE:-$(cat "$CLAUDE_PROJECT_DIR/sawmill/.active-role" 2>/dev/null || true)}"
```

Env var first → sentinel second → neither means interactive (allow all).

## Guard Allowlist

| Role | Allowed paths |
|------|--------------|
| orchestrator | `sawmill/*/TASK.md`, `sawmill/.active-role` |
| spec-agent | `sawmill/*/D[0-9]*.md`, `sawmill/*/BUILDER_HANDOFF.md` |
| holdout-agent | `.holdouts/*` |
| builder | `staging/*`, `sawmill/*/RESULTS.md`, `sawmill/*/13Q_ANSWERS.md` |
| evaluator | `sawmill/*/EVALUATION_REPORT.md`, `sawmill/*/EVALUATION_ERRORS.md` |
| auditor | `sawmill/PORTAL_AUDIT_RESULTS.md` |

Deny → `"Role [X] cannot write to [path]. Dispatch the appropriate agent."`
Unknown role → deny all.

## Sentinel Rules

- Only orchestrator writes `sawmill/.active-role`
- Means "currently active role," not session identity
- Before dispatch: write target role
- After dispatch returns: write `orchestrator`
- If stale/uncertain: delete or reset to `orchestrator`
- Workers never modify it
- `run.sh` does not use it (env vars only)
- No nested dispatch — workers must not spawn subagents

## Orchestrator Role File Addition

Add `## Dispatch Protocol` section to `.claude/agents/orchestrator.md`:
- Write target role to `sawmill/.active-role` before Agent tool call
- Write `orchestrator` back after subagent returns
- If prior dispatch was interrupted, reset sentinel before continuing
- Workers must not dispatch nested subagents
